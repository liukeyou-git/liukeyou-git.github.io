## 前言

欢迎来到我的个人博客！这是一个使用现代化技术栈构建的静态网站，旨在提供良好的阅读体验和视觉效果。本文将详细介绍博客所使用的各项技术。

## 核心技术栈

### Astro 7.x

[Astro](https://astro.build) 是一个强大的静态站点生成器，具有以下特点：

- **零 JS 默认**：只在需要时发送 JavaScript，初始加载极快
- **多框架支持**：可以混合使用 React、Vue、Svelte 等多种框架组件
- **内容集合**：内置的内容管理系统，支持 Markdown/MDX 格式
- **高性能**：自动代码分割、懒加载、预渲染等优化

### React 19

虽然 Astro 可以独立完成大部分工作，但对于需要交互的组件，我们使用了 **React 19**：

- 平滑滚动（Lenis）
- 鼠标跟随粒子效果
- 页面过渡动画（Framer Motion）
- 目录导航（Table of Contents）
- 文章点赞与分享（PostActions）
- 评论区（CommentSection）

### Tailwind CSS 4.x

样式方面采用了 **Tailwind CSS 4.x**：

- **零配置**：无需额外配置文件，通过 `@import "tailwindcss"` 直接使用
- **主题系统**：支持自定义颜色、字体、间距等设计变量
- **原子化 CSS**：高效的类名系统，快速构建界面
- **深色主题**：全站采用深色模式设计

### 动画库

为了提供丰富的交互体验，我们使用了多个动画库：

- **Framer Motion**：页面过渡动画、列表动画等复杂动画效果
- **GSAP**：高性能动画引擎，用于复杂的时间线动画
- **Lenis**：平滑滚动效果，提供丝滑的滚动体验

## 后端服务：Supabase

博客的动态功能（用户认证、评论、点赞）全部由 [Supabase](https://supabase.com) 提供。这是一个开源的 Firebase 替代方案，基于 PostgreSQL 构建。

### 主要能力

- **Auth 认证**：基于 JWT 的用户注册、登录、会话管理
- **PostgreSQL 数据库**：存储用户资料（profiles）、评论（comments）、点赞（post_likes）
- **REST API**：通过 PostgREST 自动生成 RESTful 接口
- **行级安全（RLS）**：在数据库层面控制数据访问权限
- **实时订阅**：基于 WebSocket，评论有新回复时自动更新

### 数据库表设计

| 表名 | 说明 |
| --- | --- |
| `profiles` | 用户资料（与 auth.users 关联） |
| `comments` | 文章评论，支持嵌套回复（parent_id） |
| `post_likes` | 文章点赞记录 |

为了支持以文章 slug（如 `tech-stack-intro`）作为 `post_id`，`comments` 表的 `post_id` 和 `author_id` 字段使用了 `TEXT` 类型并移除了外键约束。

### 关键实现

- **AuthContext 单例**：使用模块级单例模式，多个 `AuthProvider` 共享同一份认证状态，避免重复请求
- **maybeSingle 查询**：使用 `maybeSingle()` 替代 `single()`，优雅处理 profile 缺失的情况
- **手动创建 profile**：用户登录后若 profile 不存在则自动创建
- **分离查询 + 前端合并**：评论和作者信息分开查询，前端按 `author_id` 合并

## 项目结构

```
src/
├── components/       # 组件目录
│   ├── animations/  # 动画组件（粒子背景、页面过渡等）
│   ├── auth/        # 登录注册弹窗
│   ├── blog/        # 博客相关组件（评论、点赞、目录等）
│   ├── layout/      # 布局组件（Header、Footer）
│   └── AppRoot.tsx  # 全局 Provider 入口
├── contexts/         # React Context（AuthContext 等）
├── layouts/         # 页面布局模板
├── lib/             # 通用工具（supabase 客户端）
├── pages/           # 页面路由
├── content/         # 博客文章内容（MDX 格式）
└── styles/          # 全局样式
```

## 已实现的功能

- **用户注册 / 登录**：基于 Supabase Auth，支持邮箱密码登录
- **文章评论**：支持一级评论与回复，实时同步新评论
- **文章点赞**：登录用户可点赞，自动乐观更新
- **分享链接**：一键复制当前文章 URL 到剪贴板
- **未登录拦截**：点赞、评论、回复时自动弹出登录窗口
- **多用户发文**：注册用户可在 `/write` 编写 Markdown 文章，支持发布/草稿/编辑/删除
- **个人中心**：在 `/profile` 查看自己的全部文章（已发布/草稿）
- **用户主页**：访问 `/users/{username}` 查看某用户公开的文章列表

## 部署方式

博客部署在 **GitHub Pages** 上，通过 GitHub Actions 自动构建和部署：

1. 推送代码到 main 分支
2. GitHub Actions 自动执行构建流程
3. 将构建产物部署到 GitHub Pages
4. 访问 https://liukeyou-git.github.io 查看博客

环境变量 `PUBLIC_SUPABASE_URL` 和 `PUBLIC_SUPABASE_ANON_KEY` 配置在 GitHub Secrets 中。

## 总结

这个博客网站采用了现代化的前端技术栈 + Serverless 后端，注重性能和用户体验。前端通过 Astro 的静态生成能力保证页面加载速度；通过 React 和动画库提供丰富的交互效果；通过 Tailwind CSS 实现统一的视觉风格。后端通过 Supabase 快速构建了认证、数据库、实时订阅等能力，零运维成本即可支撑完整的 UGC 场景。
