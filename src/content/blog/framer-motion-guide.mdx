## 为什么选择 Framer Motion

在前端动画领域，Framer Motion 是我用过最舒服的动画库。它提供了**声明式的 API**，让动画逻辑和组件紧密结合，不再需要手动管理 timeline 和 keyframe。

### 核心概念：motion 组件

任何 HTML 元素加上 `motion.` 前缀就能变成可动画元素：

```tsx
import { motion } from 'framer-motion';

<motion.div
  animate={{ scale: 1, opacity: 1 }}
  initial={{ scale: 0, opacity: 0 }}
  transition={{ duration: 0.5 }}
>
  Hello Framer Motion!
</motion.div>
```

只需三行代码就实现了"从无到有缩放入场"的效果。

## Layout Animation：自动布局动画

Framer Motion 最被低估的功能。当你改变一个元素的 CSS layout 属性（如 `justify-content`、`flex-direction`），只需添加 `layout` prop：

```tsx
<motion.div layout>
  <motion.button onClick={() => setIsOpen(!isOpen)}>
    Toggle
  </motion.button>
  {isOpen && (
    <motion.p layout>Surprise content!</motion.p>
  )}
</motion.div>
```

框架会自动计算元素的位置变化并生成平滑的过渡动画——不需要写任何 CSS transition。

## AnimatePresence：元素退出动画

React 的一个痛点是组件卸载时无法做动画。`AnimatePresence` 解决了这个问题：

```tsx
import { AnimatePresence, motion } from 'framer-motion';

<AnimatePresence>
  {isVisible && (
    <motion.div
      initial={{ opacity: 0, y: 20 }}
      animate={{ opacity: 1, y: 0 }}
      exit={{ opacity: 0, y: -20 }}
    >
      I will animate out!
    </motion.div>
  )}
</AnimatePresence>
```

页面切换、弹窗、通知提示都靠这个模式。

## 实用技巧

### 1. 弹性动画

```tsx
<motion.div
  animate={{ scale: 1 }}
  initial={{ scale: 0 }}
  transition={{ type: 'spring', stiffness: 300, damping: 20 }}
/>
```

弹簧物理让动画更有"质感"。

### 2. 手势交互

```tsx
<motion.div
  whileHover={{ scale: 1.05 }}
  whileTap={{ scale: 0.95 }}
  drag
  dragConstraints={{ left: 0, right: 0, top: 0, bottom: 0 }}
/>
```

### 3. 滚动驱动

```tsx
const scrollY = useScroll();
const opacity = useTransform(scrollY, [0, 300], [1, 0]);
```

## 总结

Framer Motion 不只是"另一个动画库"，它重新思考了 React 里动画的编写方式。声明式 API + Layout Animation + AnimatePresence 这三板斧，覆盖了日常开发 90% 的动画需求。
